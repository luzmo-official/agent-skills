## ­čÜĘ Security Checkpoint (First Diagnostic)

**Before troubleshooting visual or functional issues, verify there's no underlying SECURITY misconfiguration that explains the symptom:**
- [ ] API credentials (`LUZMO_API_KEY`, `LUZMO_API_TOKEN`) are server-side only ÔÇö never in browser code, never in the failing component's props
- [ ] Embed tokens are generated by your backend via `createAuthorization`, with only `authKey` (`id`) and `authToken` (`token`) reaching the frontend
- [ ] Multi-tenant filters (`parameter_overrides`, `filters`, dataset-level `EmbedFilterGroup`, or `account_overrides`) are applied server-side, NOT in the client
- [ ] No screenshots, logs, or stack traces shared during troubleshooting expose live API keys/tokens

**If ANY checkbox is unchecked, STOP ÔÇö the bug may be a security incident.** Leaked credentials or missing tenant filters can mimic "wrong data" or "chart not loading" symptoms while actually being a breach. Fix the security issue first, then re-test.

**IMPORTANT:** Before proposing fixes, fetch the relevant developer documentation for the specific feature or API involved. Don't rely on cached knowledge - always verify current documentation shapes.

For deeper auth/embed-token diagnostics, see `core`. For tenant-isolation diagnostics, see `multitenancy`.

## Component Not Rendering At All

### Diagnostic Questions

1. **Is the component loading but invisible, or not loading at all?**
   - Not loading Ôćĺ Check component name and import
   - Loading but invisible Ôćĺ Check dimensions (Flex charts) or content

2. **Are there any console errors?**
   - Read error messages carefully ÔÇö they often point directly to the issue

3. **Is this a Flex chart or dashboard?**
   - Flex chart Ôćĺ Most likely dimension issue
   - Dashboard Ôćĺ Check auth token and dashboard ID

### Common Causes and Route To

| Symptom | Likely Cause | Route To |
|---|---|---|
| Component not loading at all | Wrong component name for framework | Fix component names Ôćĺ `data-visualization` |
| Flex chart invisible (0 height) | Missing dimensions | Fix sizing Ôćĺ `data-visualization` |
| Dashboard not appearing | Invalid authKey/authToken | Fix token generation Ôćĺ `core` |
| "Component not found" error | Package not installed or imported | Fix setup Ôćĺ `data-visualization` |
| Old component names used | Using deprecated names | Fix component names Ôćĺ `data-visualization` |

## Chart Rendering But Shows Wrong Data

### Diagnostic Questions

1. **Multiple charts on the page?**
   - Yes Ôćĺ Check for duplicate `contextId` values

2. **Recently added/modified charts?**
   - Yes Ôćĺ Verify each chart has unique `contextId`

3. **Multi-tenant application?**
   - Yes Ôćĺ Check if token filtering is properly applied (see multitenancy skill)

### Common Causes and Route To

| Symptom | Likely Cause | Route To |
|---|---|---|
| Charts showing each other's data | Duplicate `contextId` | Fix contextId Ôćĺ `data-visualization` |
| Wrong tenant's data visible | Token filtering not applied | Fix tenant isolation Ôćĺ `multitenancy` (SECURITY CRITICAL) |
| Data not updating | Stale context/cache | Fix refresh Ôćĺ `data-visualization` |

## API Errors

### Diagnostic Questions

1. **What HTTP status code?**
   - 401/403 Ôćĺ Authentication/authorization issue
   - 404 Ôćĺ Wrong endpoint or resource not found
   - 400 Ôćĺ Malformed request

2. **What operation?**
   - Authorization Ôćĺ Token generation issue
   - Resource CRUD Ôćĺ API action or structure issue
   - Query Ôćĺ Data query structure issue

### Common Causes and Route To

| Error | Likely Cause | Route To |
|---|---|---|
| "Action 'search' not found" | Using wrong action name | Fix API action Ôćĺ `core` or `resource-management` |
| 401 Unauthorized | API credentials wrong/missing | Fix credentials Ôćĺ `core` |
| 403 Forbidden (editor scenario) | Token role is `viewer` | Fix token role Ôćĺ `core` or `analytics-studio` |
| "Invalid localized string" | Using plain string instead of object | Fix localized strings Ôćĺ `data-visualization` |
| HTTP GET/PUT/DELETE not working | Using REST verbs instead of POST | Fix API architecture Ôćĺ `core` |
| 404 on API call | Wrong base URL for region | Fix region config Ôćĺ `core` |

## CORS / Cross-Origin Errors

### Diagnostic Questions

1. **Which component is throwing the CORS error?**
   - Web component dashboard in embed mode, Flex chart, or IQ widget Ôćĺ Same-domain requirement applies
   - Other API calls from your backend Ôćĺ Check region/host mismatch

2. **Is the `appServer` or `apiHost` on a different domain than the parent application?**
   - Yes Ôćĺ This is the cause. Same-domain hosting or CNAME setup is required.

### Root Cause

For **web component dashboards (embed mode), Flex charts, and charts rendered inside IQ widgets**, the `appServer` and `apiHost` must be on the **same domain** as the parent application. Pointing to `app.luzmo.com` or `app.us.luzmo.com` directly from a host like `yourapp.example.com` will cause CORS errors because the origins don't match.

If same-domain co-hosting is not possible, contact **support@luzmo.com** to arrange a CNAME record ÔÇö e.g. `analytics.example.com` Ôćĺ Luzmo app server, `analytics-api.example.com` Ôćĺ Luzmo API. This is the standard path for production deployments that need custom domains.

### Common Causes and Route To

| Symptom | Likely Cause | Route To |
|---|---|---|
| CORS error on Flex chart or web component dashboard | `appServer`/`apiHost` not same-domain as parent app | Contact support@luzmo.com for CNAME, or Ôćĺ `analytics-studio` for embed config |
| CORS error only on EU/US switch | Mixed region URLs | Fix region config Ôćĺ `core` |
| CORS error on API calls from backend | `apiHost` misconfigured server-side | Fix region config Ôćĺ `core` |

## Connection and Multi-Tenant Issues

### Diagnostic Questions

1. **Are users seeing other tenants' data?**
   - Critical security issue Ôćĺ Check tenant filtering immediately

2. **What tenant-isolation setup is in use?**
See `multitenancy` skill for the different patterns and how to check them.
 - Dataset-level (EmbedFilterGroup) Ôćĺ Check `parameter_overrides`
 - Token-level (`filters`) Ôćĺ Check filter expression
 - Connection overrides (`account_overrides`) Ôćĺ Check override structure

### Common Causes and Route To

| Symptom | Likely Cause | Route To |
|---|---|---|
| Users see all tenants' data | No filtering applied | Fix tenant isolation Ôćĺ `multitenancy` (SECURITY CRITICAL) |
| Editors can bypass filters | Using dashboard-level filters | Fix filter pattern Ôćĺ `multitenancy` (SECURITY CRITICAL) |
| Connection override not working | Wrong property name | Fix overrides Ôćĺ `multitenancy` |
| "Only one EmbedFilterGroup allowed" error | Trying to create multiple filter groups | Fix filter group Ôćĺ `multitenancy` |

## Hand Off

This skill handles diagnostics and quick fixes ÔÇö for deeper implementation, escalate.

**When to escalate to other skills:**

- WHEN the issue is auth/credentials/API-architecture (401, 403, action errors, REST verbs) Ôćĺ use `core`
- WHEN the issue is dashboard embedding (not rendering, wrong component name, sizing) Ôćĺ use `core`
- WHEN the issue is Flex chart rendering (0 height, slot errors, contextId) Ôćĺ use `data-visualization`
- WHEN the issue is ACK/editor (ACK not rendering, state out of sync, viewer can't edit) Ôćĺ use `analytics-studio`
- WHEN the issue is cross-tenant data exposure or filters not holding Ôćĺ use `multitenancy` (SECURITY CRITICAL)
- WHEN the issue is IQ answer quality, IQ license, or IQ component loading Ôćĺ use `ai-analytics`
- WHEN the issue is data not loading, dataset/connection problems, createData errors Ôćĺ use `data-integration`
- WHEN the issue is theming (dark mode container white, brand colors not applying) Ôćĺ use `theming`
- WHEN the issue is script-side API errors (search/get confusion, delete safety) Ôćĺ use `resource-management`

**This skill does NOT cover:**

- Building new features from scratch (use the relevant feature skill directly)
- Long-form architectural decisions (this is for fast diagnosis only)

## Emergency Debugging Checklist

When completely stuck, work through this checklist:

1. Ôťů Check browser console for errors
2. Ôťů Verify component names match framework (see data-visualization skill)
3. Ôťů Confirm API credentials are correct and server-side only
4. Ôťů Check that embed tokens are being generated and passed correctly
5. Ôťů Verify `apiHost` and `appServer` match your Luzmo region ÔÇö and for Flex charts, web component dashboards, and IQ widgets, they must also be on the **same domain** as the parent app (if not, a CNAME request sent to support@luzmo.com is required)
6. Ôťů Ensure all user-facing text uses localized objects: `{ en: "..." }`
7. Ôťů Check that `contextId` is unique for each chart
8. Ôťů For Flex charts: verify dimensions set on both container and component
9. Ôťů For API calls: confirm using POST with `action` in request body
10.Ôťů For multi-tenant: verify filtering is applied in authorization token (server-side)

## Still Stuck?

If issue persists after trying relevant fixes:

1. **Fetch the specific documentation** for the component/API you're using
2. **Read error messages carefully** ÔÇö they often contain the solution
3. **Check for recent changes** ÔÇö what was the last thing that worked?
4. **Isolate the issue** ÔÇö does it happen with a minimal example?
5. **Contact Luzmo support** at support@luzmo.com with:
   - Clear description of issue
   - Error messages (screenshots/text)
   - Minimal reproducible example
   - Browser/environment details
